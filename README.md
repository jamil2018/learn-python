# Learn Python by Building

This workspace is a practice-first Python track for someone who already knows TypeScript and wants Python as a second working language.

The rule of the course is simple: solve problems first, read notes only when a problem exposes a gap.

## How To Use This

1. Work through the module folders in [modules](modules).
2. Read that module's `notes.md` when you need the core ideas.
3. Inside each module, open the topic folders in order.
4. Each problem has its own folder with a single `problem.md`.
5. Write your solution file next to that problem's `problem.md`.
6. After the problems, build the project in that module's `99-milestone-project/` folder.
7. Keep a small `journal.md` with mistakes, surprises, and syntax you had to look up.
8. Do not optimize early. Make it correct, make it readable, then make it nicer.

## Suggested Weekly Rhythm

- **Day 1-4:** Solve 8-15 exercises.
- **Day 5:** Re-solve 3 older exercises from memory.
- **Day 6-7:** Work on the current module project.

If a problem feels too easy, add constraints: no loops, no built-ins, handle invalid input, write tests, support a file input, or make it usable from the command line.

## What To Avoid

- Do not binge-watch tutorials before attempting the exercises.
- Do not copy solutions from the internet.
- Do not skip writing small programs because they feel obvious.
- Do not treat Python as TypeScript without types. Let the language teach you its own style.
